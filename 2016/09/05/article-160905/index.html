我的前端开发环境——VS Code篇(一)

发表于 2016-09-05   |   分类于 前端

简介

VS Code是微软出品的一款轻量级Web集成开发环境，在运行的流畅性及功能的可扩展性这两个方面的表现都十分出色，尤其对于前端开发，其JS及CSS的智能提示集成带来了许多便利之处，同时还不得不提它自身对git的支持以及代码的调试功能。不仅支持git的基本操作，可以直接查看项目文件的开发记录，经过配置还可以对Node.js等进行代码调试，可谓强大。但相对于Sublime Text以及Atom这两个比较成熟的Editor，VS Code还是存在不少不足之处的，比如：非js文件无法支持代码智能提示，html文件中css提示需要第三方插件支持等。但这些缺点并不阻碍我对这款Editor的喜爱，接下来具体介绍一下我个人的配置。

配置

界面

  • 字体选择

VS Code支持Windows，Linux以及Mac三个平台，界面风格都比较一致，唯一区别比较大的方面在于windows的字体渲染上，比起Mac的还是逊色许多，所以挑选一款美观的源码字体对开发的舒适度还是有蛮大改观的。这里我推荐Hack以及Source Code Pro这两款字体，其中Source Code Pro是来自Adobe公司的一款开源字体。我个人比较喜欢Hack，其在windows下VS Code的显示效果如下图所示。这两者都是挺不错的选择！

插件推荐

VS Code本身已经集成了emmet前端神器，所以不需要额外安装插件了，以下插件可以根据开发需求自行添加：

  1. HTML Snippets

    提供html模板及代码片段

  2. Beautify

    美化代码(缩进及格式)

  3. Trailing Spaces

    去除代码中多余的空格

  4. Align

    调整代码缩进

  5. HTML CSS Support

    在HTML中添加CSS代码提示支持

  6. Rainbow Brackets

    为小括号，大括号等添加彩虹色匹配

  7. ESLint

    javascript代码规范检查(提供ES6及JSX支持)

  8. markdownlint

    markdowm语法规范检查

  9. Git History

    查看git历史记录

  10. Autoprefixer

    自动为CSS代码添加浏览器前缀

  11. CSScomb

    自动调整CSS属性顺序

  12. vscode-icons

    为资源管理器的文件树提供图标前缀

  13. AutoFileName

    自动文件路径补全
