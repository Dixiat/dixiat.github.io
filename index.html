更换系统或重装系统后hexo博客的转移和恢复

发表于 2016-07-17   |   分类于 技术宅

更换电脑或重装系统后hexo博客需要在新的电脑或系统下进行重新配置才能恢复使用，恢复工作如下：

一、 安装git及node.js

恢复hexo博客的使用，基本的工具和环境是必需的。
git用于本地文件与github的同步，node.js提供npm工具安装hexo插件。

二、 配置SSH Keys

让本地git项目与远程github建立联系需要SSH Keys的认证，所以需要在本地重新配置SSH Keys。

1. 检查本地SSH Keys的设置

在配置SSH Keys之前，需要检查电脑上是否存在现有的SSH Keys：

如果存在id_rsa.pub或id_dsa.pub，则直接将SSH Key添加到Github中，若本地不存在SSH Keys，则进入下一步——生成SSH Key。

2. 生成SSH Keys

  • 生成public/private rsa key pair

在命令行中输入以下语句：

Windows下默认会在C:\Users\用户名.ssh路径下生成id_rsa和id_rsa.pub两个文件。

  • 输入passphrase（本步骤可以跳过–回车）

这个密码用于项目提交时的认证，为空时提交项目不需要输入。这个设置是防止别人未经允许向你的项目提交内容。

3.添加SSH Key到Github

完成本机的SSH Key生成后，需要将其添加到Github上，以完成SSh链接的设置。

  • 打开 C:\Users\用户名.ssh\id_rsa.pub 文件（公钥）并复制文件中的内容

  • 登录Github账号，选择头像下 Setting ==> SSH and GPG keys ==> New SSH Key

  • 将复制的密钥内容粘贴至Key 文本框中并设置一个Title，点击Add key即完成
